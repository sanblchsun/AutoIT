

Include_Helper - утилита показывает Include UDF которые необходимы скрипту. А также недостающие и лишние, если в скрипте уже имеются Include

Как всё начиналось: идея sleepydvdr, тотальная переделка от AZJIO)
http://www.autoitscript.com/forum/topic/130468-constants-helper/page__view__findpost__p__908064

Обычно старые скрипты при старте начинают ругаться на отсутствие значений переменных, после добавления (поиском в UDF-скриптах Include) происходит запрос следующей и так бесконечно долго пока не будут добавлены все переменные. Чтобы сделать быстро, обрабатываем скрипт в Include_Helper и получаем список недостающих UDF. Список переменных создаётся при первом старте.

Текущие недостатки:
1. Скрипт скрыто завершается, если нет ни одной функции или переменной в скрипте, хотя случай редкий, но всё же вызывает недоумение (можно сделать сообщение). (некритично)
2. Не проверяется текущий каталог, из-за этого такие UDF попадают в список лишних. (некритично)

Текущие плюсы:
1. Высокая скорость обработки
2. Работает без зависимости от SciTE
3. Работает с ком-строкой, иначе предлагает открыть скрипт-файл
4. Исключает вложенные Include, то есть добавляет минимально возможное количество Include
5. Можно посмотреть лог срабатывания (причины, по которым добавлена Include)

Возможно плюс, возможно минус (баланс между скоростью и практичностью)
1. Недобавлены проверка функций вызваных с помощью Call. Причиной тому, что функции UDF обычно не вызываются с помощью Call. Этот способ используется для вызова генерируемых в диапазоне имён функций, но функции UDF уникальны.

Обновления
Добавляет @LF, если вместо него используется @CR, который не является переносом строк для регулярного выражения с ключом m
Закомментированный текст не попадает в обработку, в итоге в список не попадают необязательные для скрипта UDF.
