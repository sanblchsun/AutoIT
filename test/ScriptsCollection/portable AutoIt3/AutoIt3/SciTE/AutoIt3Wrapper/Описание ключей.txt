
AutoIt3Wrapper используя ключи позволяет автоматически применить некоторые параметры при компиляции.
Внешние ключи AutoIt3Wrapper отменяются внутрении ключами скрипта, или другими словами параметры AutoIt3Wrapper указанные в скрипте являются более приоритетными.

Старт AutoIt3Wrapper в CUI, с возможностью указать параметры.
/ShowGui

путь к скрипту для компиляции
/in "%l"

путь к получаемому скомпилированному скрипту
/out "C:\my_program.exe"

стандартная иконка
/icon "C:\AutoIt3\Icons\au3.ico"

сжатие, максимальный - 4, минимальный - 0
/comp 4

Пропустить сжатие UPX.
/nopack

Включить сжатие UPX (по умолчанию)
/pack

Компилировать в Ansi для использования в Win9x
/ansi

Компилировать в Unocode (по умолчанию)
/unicode

Компилировать для x64 OS.
/x64

Выходное приложение консоль
/console Change output program to CUI

Выходное приложение GUI (по умолчанию)
/Gui

Указать каталог AutoIt3
/autoit3dir "C:\AutoIt3"


/prod
/beta
/UserParams $(1) $(2) $(3) $(4)
/run - просто стартует скрипт
/ErrorStdOut - ничего кроме компиляции
/AU3Check - ничего не сделал совсем

Примеры

/run /autoit3dir "C:\Program Files\AutoIt3_3.3.0.0" - комбинация позволяет стартовать скрипт используя другую версию AutoIt3.
/autoit3dir "C:\AutoIt3" - несколько пунктов контекстного меню с указанием другого каталога позволяют компилировать скрипт разными версиями AutoIt3.
Если сделать вложенное меню в Win7, то это даст возможность удобно работать с разными версиями программ.
