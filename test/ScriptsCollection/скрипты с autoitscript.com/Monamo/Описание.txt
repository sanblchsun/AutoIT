Скрипт копирования пути в буфер обмена
http://www.autoitscript.com/forum/index.php?showtopic=57144

Смысл такой: при клике срипт/программа ничего не делает, ей нужно указать ключ для установки. Получив ком-строку она добавит в реестр ком-строку передачи файла скрипту. который отправит в буфер полный путь.

Подобную утилиту я тоже делал, но с чуть большим функционалом. В контекстное меню добавляется пункты "Управление файлом", "Управление каталогом". Выбрав пункт получаем менюшку с доступными функциями, а точнее:
Для каталога "Передать список файлов в папке в буфер обмена", "Открыть ком-строку", "Дать доступ ка папке", Вызов сканера просмотра объёма", "Удалить файлы из папки указав расширение(я)".
Для файлов "Копировать линк в буфер", "Открыть ассоциированную программу", "Дать доступ к файлу".
Только авто-инсталятор нужно добавить в скрипт.

Скрипты перехватывают F2 и выделяют часть имени файла без расширения, для переименования.
http://www.autoitscript.com/forum/index.php?showtopic=88903
SmartRename for XP.au3 - подправил проблему Ctrl+C (Ctrl+insert) и выделение сделал справа. Присутствует восстановление буфера, так как имя копируется через буфер.
SmartRename for XP (API).au3 - скрипт далее по теме, поудобнее, но требует кучу UDF, обработал в Au3PP, теперь компактно.
Ещё бы при создании файла также выделялось имя файла без расширения...
